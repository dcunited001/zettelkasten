** Usage

*** Environment

[[https://github.com/ipspace/NetOpsWorkshop][ipspace/NetOpsWorkshop]] focus on building the VM environment for containers to
run on. See Vagrantfile there and in the netlab project. It doesn't mention
netlab per se, but odds are the network deps there are sufficient (or close)

**** Devcontainer

Blog strongly recommends to run in a devcontainer, hosted on ghcr.io
[[https://github.com/ipspace/netlab/pkgs/container/netlab%2Fdevcontainer][netlab/devcontainer]]. There isn't a =Dockerfile= likely bc of the difficult/issues
that arise with Docker-in-Docker or Docker-to-Docker.

+ See [[https://github.com/ipspace/netlab/blob/dev/.devcontainer/devcontainer.json][.devcontainer/devcontainer.json in netlab]]. user is =vscode=. 

+ [[https://containers.dev/implementors/json_schema/][devcontainer.json.schema]]
  - workspaceMount :: the =--mount= param for docker run. it defaults to
    =/workspace/$project=... (i really don't actually know. i live on mars)
  - workspaceFolder :: path to the workspace dir inside the container
