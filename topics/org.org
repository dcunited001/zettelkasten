*** Quickstart Manual
All of the contents of the quickstart are absolutely essential to engrain in
habits as early as possible. If you haven't done that, then you'll want to
return to it occasionally. It's a bit challenging when you lack the experience
to clarify the dao of each feature

I've unfortunately reinforced the wrong muscle memory, but there are essential
things like:

+ magically understanding the appropriate folder/file structure that works for
  you, along with the capture/refile "pipelines" to push things through
  - also ... actually using capture/refile. these templates should be fairly easy
    to extend.
  - you should expect the top-level tags in each type of =org-mode= and =org-roam=
    doc to be similar. really helps with memory to simplify movement/actions.
  - a good approach (not well-tested) to should have a generic set of files
    (like =todo.org= and =notes.org=)
  - picking an approach to intra-document and extra-document refiling
+ use the _org-link_ functionality (i don't really capture with the custom URI
  protocol)
  - org-insert-link C-c C-l :: use =org-store-link=. i was a little confused at
    first by how it managed the completion entries.
  - org-store-link :: This is =C-c n l= in Doom Emacs.
    - Again, use this instead of um not. Emacs and Org are pretty good at
      inferring link type based on cursor position, mode and file type. So
    - this will basically run =org-id-get-create= on the current header
  - org-insert-last-stored link :: convenient.
  - orgit :: stores a link to the document
    - =orgit-rev:/data/org/roam/::84216e9ed9da45eac03678953587cbe4bd2a2020=
    - =~/org/roam/ (magit-rev 84216e9)=
+ how to extend =org-todo-keywords= and understanding why you basically never want
  to do that (the org equivalent of bitrot).
  - it's better to just reappropriate & remap the extant =TODO= keywords to your
    document's purpose & its specific workflow requirements.
  - the data structures that back them -- number of values, the gui controls
    that they loosely map too -- are highly generic.
  - =org-todo-keywords= basically expect your emacs config to set them ... or the
    document itself (which gives warnings when you load it). Tags on the other
    hand are simple to extend.

There are about 50 of these things (mostly simple!) and they were easy to forget
after my first run-through of the org quickstart.
