* What's the difference between virtual routers and virtual systems?

#+begin_quote
On PaloAlto: the scope/isolation of network functions provided by each *box*
abstraction. 
#+end_quote

VRouters use VRF & VSystems use more comprehensive namespace isolation

#+begin_quote
VRouters: independent routing/forwarding instances; separate routing tables,
interfaces, and policies. Segments networking functionality only.

VSystem (VSY): self-contained firewall instance with separate security policies,
user accounts, logs, etc. Isolation is sufficient for multi-tenancy.
#+end_quote

"Everything is a box with little boxes. Some boxes are heavy, some are like
cattle & others like pets; some poke through the walls of other boxes. Even
network layers & protocol abstractions are like boxes. VRFs are a type of box
that feel flipped inside out. Don't mix the wrong boxes.

"Often, if there isn't physical separation, the _distance_ through network
isolation is probably diminished. Distance/space via network abstractions
creates increased cost/complexity to attack ... until it increases
cost/complexity to defend. Very small networks have limited capacity for netflow
collection or planes (mgmt/etc) ... and lack flexibility to change configuration
(e.g. migrate routers to new switch stack; migrate addressing for
firewall/router)"
