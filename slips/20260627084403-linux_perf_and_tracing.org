** New Developments
*** eBPF

+ [[https://ebpf.io/what-is-ebpf][What is eBPF?]]
  - BPF :: Berkeley Packet Filter
    * BPF has provided packet inspection capabilities to Linux, but has recently
      been upgraded to improve performance, programmability and
      kernal/component(?)/software interoperability.
    * It does this with LLVM and JIT compiler magic. The performance
      improvements were likely required.
    * Cilium requires BPF capabilities from Linux 4.8.0+.
  - XDP :: eXpress Data Path

#+begin_quote
XDP is a further step in evolution and enables to run a specific flavor of BPF programs from the network driver with direct access to the packet's DMA buffer. This is, by definition, the earliest possible point in the software stack, where programs can be attached to in order to allow for a programmable, high performance packet processor in the Linux kernel networking data path.
#+end_quote

+ see the [[https://docs.cilium.io/en/stable/bpf/][BPF and XDP reference guide]] to understand how the components are used in the Cilium
