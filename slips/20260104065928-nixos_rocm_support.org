+ [[https://github.com/ROCm/composable_kernel/blob/develop/TERMINOLOGY.md][Composable Kernel Terminology]]
  - "A CK pipeline orchestrates the sequence of [resource utilization]
    operations for a kernel ... and consists of two core components"
  - Problem :: "What to compute: input/output shapes, data types, math ops
    (GEMM, convolution)"
  - Policy :: "Memory access patterns, hardware-specific optimizations"
  - So it connects intent to resources, while enforcing rules. It "simplifies"
    the process of declaring operations, managing resources and making data
    about load available to facilitate constraints ... i think
