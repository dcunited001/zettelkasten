Run =git stack= to see the trees. Then =git stack --dump-config -= to check the config

| Command                                     | Purpose                                                                    |
|---------------------------------------------+----------------------------------------------------------------------------|
| ~git config --global set stack.stack current~ | Makes =git-stack= commands safer for random clones until they're configured.                          |
| ~git stack --protect develop~                 | Protect additional branches. On a team, you'll need branch naming patterns |
| ~git config set stack.stack dependents~       | Limit the default scope of ~git-stack~ commands                              |

* Stacking Modes

| Stack Type  | Selector                 | Desc                                                                                        |
|-------------+--------------------------+---------------------------------------------------------------------------------------------|
| current     | Branches in ~BASE..HEAD~   | Fairly useless                                                                               |
| dependents  | Branches in ~BASE..HEAD..~ | Runs up stack, listing refs depending on this branch/stack. Safe. Mostly useful if on a team |
| descendants | Branches in ~BASE..~       | Traces up stack & down to their common ancestor. Limited by =protect-commit-age=. Less safe.   |
| all         | Show all branches        | Not really safe/useful for automation.                                                       |

The =stack.stack= config will require using =git stack -s all= to get the pretty
tree back. This limits the scope of automation that =git-stack= can perform.

Next, ensure that each branch you care about has a =ref= set in
=branch.featname.merge=.

* Command

~git log --graph --all --oneline --decorate main..HEAD~

The main =git-stack= command is functionally equivalent to the above. It also
assumes user intent based on context.

|-----+----------------+---------+--------------------------------+----------------------------------------------------------------|
| Opt | Option         | Default | Values                         | Desc                                                           |
|-----+----------------+---------+--------------------------------+----------------------------------------------------------------|
| ~-r~  | ~--rebase~       |         |                                | Rebase the selected stacks                                     |
|     | ~--pull~         |         |                                | Pull the parent branch and rebase onto it                      |
|     | ~--push~         |         |                                | Push all ready branches                                        |
| ~-s~  | ~--stack~        | all     | current,dependents,descendants | Which branch stacks to include                                 |
|     | ~--base~         | ~$branch~ |                                | Branch to evaluate from. Default: most recent protected branch |
|     | ~--onto~         | ~$base~   |                                | Branch to rebase onto                                          |
|     | ~--fixup~        | ignore  | move,squash                    | Action to perform with fixup-commits                           |
|     | ~--repair~       |         |                                | Repair Diverging branch                                        |
|     | ~--format~       | graph   | silent,list,debug              |                                                                |
|     | ~--show-commits~ |         | none,unprotected,all           |                                                                |
|     | ~--protected~    |         |                                | Show protected branches                                        |
|     | ~--protect~      |         | ~$branch~                        | Protect ~$branch~'s that match =gitignore= syntax.                 |
|     | ~--dump-config~  |         | ~$output_file~                   | Use =-= for stdout                                               |
|-----+----------------+---------+--------------------------------+----------------------------------------------------------------|
** Actions

| Action   | Alternative                                               | Benefit                                                            |
|----------+-----------------------------------------------------------+--------------------------------------------------------------------|
| ~--rebase~ | ~git rebase -i --autosquash master~                         | No need to manually select the base                                |
| ~--repair~ | ~repo forall~ or raw bash                                   | Update dependent stacks of new commit(s) in their parent stack     |
| ~--push~   | ~git push --set-upstream --force-with-lease origin $branch~ | Prevents pushing branches if multi-stack or if include ~WIP~ commits |

=git-stack --push= doesn't prevent pushing =fixup!= commits. to avoid that, use
the =commited= CLI tool

** Subcommands

Auto-rebase will rebase child commits/branches, given the assumed stack logic
for a scope set by =-s $stack_type= or by =git config get stack.stack=.

Most commands don't offer =-s=, so choosing the correct project config is
necessary.

| Subcommand | Alternative                       | Benefit                                |
|------------+-----------------------------------+----------------------------------------|
| sync       | ~git pull --rebase upstream main~   |                                        |
| next       | ~git stack && git checkout $ref~    | No need to copy =$ref=                   |
| previous   | ~git stack && git checkout $ref~    |                                        |
| reword     | ~git commit --amend~                | Auto-rebase                            |
|            | ~git rebase -i $ref~ then setting ~r~ | Auto-rebase. Less steps                |
| amend      | ~git commit --amend --no-edit~      |                                        |
| run        | ~repo forall~                       | You probably don't have =repo= installed |

Running =git-stack --rebase= will squash the ~fixup!~ commits, depending on
=stack.auto-fixup=.

*** Global

=git-stack sync= only takes these options, which apply to all subcommands.

| Opt | Option        | Default | Values       | Desc                       |
|-----+---------------+---------+--------------+----------------------------|
| ~-n~  | ~--dry-run~     |         |              | Don't actually switch      |
|     | ~--color $when~ | auto    | always,never | Controls when to use color |
| ~-v~  | ~--verbose...~  |         |              | Increase logging verbosity |
| ~-q~  | ~--quiet...~    |         |              | Decrease logging verbosity |
| ~-h~  | ~--help~        |         |              | Print help                 |
|-----+---------------+---------+--------------+----------------------------|

*** =git-stack run=

| Opt | Option         | Desc                                  |
|-----+----------------+---------------------------------------|
| ~-s~  | ~--switch~       | Switch to the first commit that fails |
|     | ~--no-fail-fast~ | Keep going                            |
|-----+----------------+---------------------------------------|
*** =git-stack {reword,amend}=

=git-stack amend -m $message [OPTIONS] [REV]=

=git-stack reword -m $message [REV]=

Only =-m= is valid for reword.

| Opt | Option        | Value    | Desc                         |
|-----+---------------+----------+------------------------------|
| ~-a~  | ~--all~         |          | Commit all changed files     |
| ~-i~  | ~--interactive~ |          | Interactively add changes    |
| ~-e~  | ~--edit~        |          | Force edit of commit message |
| ~-m~  | ~--message~     | ~$MESSAGE~ | Commit message               |
|-----+---------------+----------+------------------------------|

*** =git-stack {next,previous}=

| Opt | Option      | Desc                                   |
|-----+-------------+----------------------------------------|
| ~-b~  | ~--branch~    | Jump directly to the previous branch   |
|     | ~--stash~     | Stash prior to switch                  |
|     | ~--oldest~    | On ambiguity, select the oldest commit |
|     | ~--protected~ | Traverse across protected commits      |
|-----+-------------+----------------------------------------|
